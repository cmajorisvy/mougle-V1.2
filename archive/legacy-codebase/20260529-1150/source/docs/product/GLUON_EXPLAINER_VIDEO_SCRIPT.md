# Gluon Explainer Video Script

## Purpose
Short product explainer text for Illustrator/motion graphics/onboarding use.

## Audience
Public users, contributors, staff, and admins.

## Runtime Target
60-90 seconds.

## Tone
Simple, clear, calm, non-financial, trust-first.

## Scene 1 - What Is Gluon?
Visual direction:
Show a simple contribution card with a safe display ID.

On-screen text:
"Gluon is a contribution identity system."

Narration:
"When someone contributes knowledge, evidence, review work, or useful signals to Mougle, the system can create a Gluon ID to identify that contribution."

## Scene 2 - What a Gluon ID Shows
Visual direction:
Show example cards:
Genesis ID: GEN-000184
Packet ID: PKT-000088
Status: Admin-reviewed

On-screen text:
"ID. Type. Status. Visibility."

Narration:
"A public Gluon view only shows simple information: the ID, the type of contribution, the review status, and whether it is public-safe, private, or admin-reviewed."

## Scene 3 - What Gluon Is Not
Visual direction:
Show crossed-out words: cash, token, shares, payout.

On-screen text:
"Not cash. Not a token. Not shares. Not a payout promise."

Narration:
"Gluon IDs are contribution records, not cash, tokens, shares, or payout promises."

## Scene 4 - Gluon Types
Visual direction:
Show five simple labeled cards.

On-screen text:
"Genesis. Genome. Signal. Packet. Legacy."

Narration:
"Genesis marks an original or first-source contribution. Genome connects to agent learning patterns. Signal marks a smaller usefulness or trust marker. Packet identifies a reviewed knowledge bundle. Legacy marks an imported or historical contribution record."

## Scene 5 - Public vs Admin Views
Visual direction:
Split screen.
Left: public card with ID only.
Right: admin dashboard with analysis labels.

On-screen text:
"Public sees identity. Admin sees analysis."

Narration:
"Public views stay simple and safe. Admin tools keep deeper analysis private, including governance review, trust impact, UES movement, GVI references, evidence links, and risk flags."

## Scene 6 - How Users Use It
Visual direction:
Show a user referencing a Packet ID in a contribution timeline.

On-screen text:
"Use Gluon IDs to reference reviewed contributions."

Narration:
"Users can use Gluon IDs to track and reference reviewed contributions without seeing internal scoring or financial-looking estimates."

## Scene 7 - How Admins Use It
Visual direction:
Show admin review queue.

On-screen text:
"Admins use Gluon for review, quality, and governance."

Narration:
"Admins and staff can use Gluon analysis to review contribution quality, check evidence links, understand risk, and keep Mougle's knowledge economy safe."

## Scene 8 - Closing
Visual direction:
Show a clean Gluon Passport card.

On-screen text:
"Gluon: proof of contribution, safely displayed."

Narration:
"Gluon helps Mougle recognize contribution history while keeping economy analysis, governance analysis, and internal scoring safely controlled."

## Short Version - 30 Seconds
Narration:
"Gluon is Mougle's contribution identity system. A Gluon ID helps identify reviewed knowledge, signals, packets, or learning patterns. Public views show only safe details like ID, type, status, and visibility. Deeper metrics such as GVI, UES, trust impact, and risk analysis stay inside admin tools. Gluon IDs are contribution records, not cash, tokens, shares, or payout promises."
