Start R3 — Admin-only R3F Preview Sandbox

Goal:
Create the first safe R3F preview sandbox for Mougle Production House / 3D-4D development.

This is only a sandbox.
Do not integrate with Production House packages yet.
Do not load real private assets yet.
Do not add upload flows.
Do not add schemas or migrations.
Do not call providers.
Do not enable render/export/live/Unreal/4D hardware behavior.

Allowed implementation:
1. Add one admin-only route:
   /admin/r3f-preview-sandbox

2. Add one page/component:
   client/src/pages/admin/R3FPreviewSandbox.tsx
   or under an existing admin/production-house folder if that is the current convention.

3. Add a lazy-loaded R3F canvas component:
   client/src/components/production-house/r3f/ProductionCanvasSandbox.tsx

4. Scene contents:
   - simple grid/floor
   - simple camera
   - ambient/directional lights
   - one or two primitive meshes only
   - no external model loading yet
   - no texture loading yet
   - no GLB/GLTF yet
   - no Unity iframe yet

5. Safety UI:
   Add visible badges:
   - Admin preview only
   - No public URL
   - No signed URL
   - No provider calls
   - No render execution
   - No Unreal execution
   - No 4D hardware
   - No publishing

6. Performance guard:
   - lazy-load Canvas
   - cap DPR, e.g. dpr={[1, 1.5]}
   - use frameloop="demand" if possible
   - keep scene small
   - add fallback if WebGL unavailable
   - no setState inside useFrame
   - no heavy animation loop

7. Dashboard link:
   Add a card/link under 3D/4D/Unreal or Production House:
   label: R3F Preview Sandbox
   route: /admin/r3f-preview-sandbox
   status: Dry run / Admin only
   description: Browser-only R3F sandbox for safe 3D preview experiments. No assets, no render, no public output.

8. Tests/checks:
   - TypeScript check for changed files
   - npm run build
   - route grep confirms /admin/r3f-preview-sandbox exists
   - zero provider/env secret references from the sandbox
   - zero schema/migration changes
   - zero backend route changes
   - zero publicUrl/signedUrl/realSendAllowed/executionEnabled fields added or enabled
