Part A — /admin/ai-jobs query-param filters

Update /admin/ai-jobs so it reads URL query params on load and applies them to the existing filters.

Support query params:

- status
- jobType
- origin
- lockedBy
- requestedByUserId
- requestedByAdminId
- since
- until

Examples:

/admin/ai-jobs?status=failed
/admin/ai-jobs?status=running
/admin/ai-jobs?status=failed&since=24h
/admin/ai-jobs?lockedBy=worker-1
/admin/ai-jobs?origin=inhouse

Behavior:
- initialize filter controls from query params
- when filters change in the UI, update the URL query params
- preserve manual refresh behavior
- handle unknown/invalid params gracefully
- support relative since values if easy:
  - 24h
  - 7d
  - 30d
  Otherwise only support ISO date strings and ignore unsupported relative values safely.
