This is still an admin-only development preview.

executionEnabled must remain false.

5. Tooltips

Add tooltips to every major button, selector, status badge, and section heading.

Tooltips should explain:
- what the function does
- whether it is mock, draft, dry-run, or contract-only
- whether it calls an external provider
- whether it can affect Unreal
- whether it can affect 4D hardware
- whether it can publish anything
- what safety restrictions apply

Examples:

Production Wizard tooltip:
“Guides root-admin through creating a draft/internal production package with room, avatars, media package, 4D cue suggestions, and preview. Does not render, publish, execute Unreal, or trigger 4D hardware.”

3D/4D Room Generator tooltip:
“Creates draft/internal room specifications for newsroom, podcast, debate, hall, and cinema environments. This does not create a live Unreal level or import assets.”

Avatar Creator tooltip:
“Creates draft/internal avatar and accessory specifications. This does not generate MetaHuman rigs, lip-sync, or live avatar animation yet.”

Preview Studio tooltip:
“Admin-only visual mock preview. Not an Unreal render. Not a published video. No 4D hardware is triggered.”

Unreal Dry-Run tooltip:
“Sends dry-run-only validation payloads to the Unreal bridge. No level load, render, asset import, Sequencer start, or Movie Render Queue execution.”

4D Hardware Sandbox tooltip:
“Validates and simulates 4D cue payloads. No DMX, OSC, UDP, MIDI, serial, relay, fog, wind, scent, lighting, or motion-seat hardware command is sent.”

Publishing tooltip:
“Publishing is disabled. YouTube upload, social posting, live streaming, and autonomous publishing remain blocked.”
