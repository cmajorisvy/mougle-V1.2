Let admins share a saved broadcast view with the whole team.

Goal:
Admins should be able to create a saved broadcast view/filter and mark it as shared so all admin team members can see and use it.

Context:
There is already an admin_filter_views table or related saved-view system. If a migration prompt asks whether admin_filter_views is new or renamed, choose the default create table option unless the table already exists.

Scope:
- Add team-shared saved broadcast views.
- Do not make views public.
- Do not expose views outside admin.
- Do not change publishing, newsroom safety gates, Unreal, 4D, or Production House execution behavior.

Requirements:

1. Data model

Extend or use admin_filter_views with fields like:

- id
- ownerUserId
- name
- description nullable
- viewType:"broadcast" or existing compatible type
- filters JSON
- columns JSON optional
- sort JSON optional
- isShared:boolean default false
- sharedWithTeamAt nullable timestamp
- createdAt
- updatedAt

If the table already supports these concepts, reuse existing fields instead of duplicating.

2. Permissions

Rules:
- Root-admin can create, edit, delete, and share any broadcast view.
- Admins can create private views.
- Admins can share their own views if permission allows.
- Shared views are visible to all authenticated admins.
- Non-admin users cannot read shared admin views.
- Public/anonymous users cannot read any admin views.
- A normal admin cannot edit/delete another admin’s shared view unless existing permission policy already allows it.

3. Backend routes

Add or extend routes:

GET /api/admin/broadcast-views
POST /api/admin/broadcast-views
PATCH /api/admin/broadcast-views/:id
DELETE /api/admin/broadcast-views/:id
POST /api/admin/broadcast-views/:id/share
POST /api/admin/broadcast-views/:id/unshare

All mutating routes require admin auth and CSRF.

GET should return:
- user’s private views
- views shared with the team

4. UI

In the broadcast/admin dashboard view, add:

- Save View button
- Saved Views dropdown
- Share with Team toggle/button
- Shared badge
- Owner label
- Last updated timestamp
- Delete button where allowed
- Rename/Edit view where allowed

If a view is shared, display:

“Shared with team”

5. Audit events

Add audit events:

- broadcast_view.created
- broadcast_view.updated
- broadcast_view.deleted
- broadcast_view.shared
- broadcast_view.unshared
- broadcast_view.loaded

Audit details must not include secrets.

6. Safety

Saved filters/views must not be able to:
- trigger publishing
- trigger YouTube upload
- trigger social posting
- trigger live Unreal execution
- trigger real 4D hardware
- bypass approval gates
- expose private provider metadata
- expose secrets or tokens

7. Tests

Add tests proving:

- anonymous users cannot list views
- non-admin users cannot list views
- admin can create private saved view
- admin can share own view
- shared view is visible to another admin
- private view is not visible to another admin
- root-admin can manage shared views
- normal admin cannot edit/delete another admin’s shared view if not allowed
- mutating routes require CSRF
- saved filters cannot contain publish/YouTube/social/live-Unreal/4D execution actions
- audit events are written
- no secrets are returned

8. Run:

- npm run check
- relevant saved-view/admin-filter-view tests
- relevant broadcast dashboard tests if present

Do not enable publishing.
Do not enable YouTube/social posting.
Do not enable live Unreal execution.
Do not enable real 4D hardware.
Do not weaken approval gates.
