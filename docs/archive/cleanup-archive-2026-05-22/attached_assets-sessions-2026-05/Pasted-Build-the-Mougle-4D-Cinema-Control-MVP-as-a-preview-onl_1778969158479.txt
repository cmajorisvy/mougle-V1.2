Build the Mougle 4D Cinema Control MVP as a preview-only extension of the existing Mougle Phase 1B verified newsroom pipeline.

Context:
The current Mougle repo already includes:
- Phase 1B verified newsroom architecture/specs
- clustering and claim extraction services
- NewsroomDataPackage builders
- canonical RenderManifest contract
- admin dry-run preview routes
- persistent storage readiness checks
- FFmpeg safety hardening
- environment validation
- shutdown registry
- no-publish safety envelope
- root-admin gating
- CSRF protection on mutating API routes

Important:
This is NOT a greenfield app.
Do not replace the existing Mougle architecture.
Extend the existing admin/preview system safely.

Goal:
Create a preview-only Mougle 4D Cinema Control MVP for:
1. AI newsroom scenes
2. podcast room scenes
3. AI avatar presenter scenes
4. Unreal Engine scene manifest export
5. 4D cue manifest export
6. admin approval before any future Unreal or 4D hardware trigger

This MVP must generate JSON manifests only.
It must not execute Unreal commands or 4D hardware triggers in production yet.

Hard constraints:
- Do not modify shared/schema.ts.
- Do not import shared/newsroom-schema.ts into shared/schema.ts.
- Do not run db:push.
- Do not add migrations.
- Do not enable public publishing.
- Do not enable YouTube, social, RTMP, or live streaming.
- Do not execute FFmpeg, Remotion, Unreal Engine, avatar-video-render-service, or render workers.
- Do not send real commands to Unreal unless explicitly in dry-run/mock mode.
- Do not send real 4D hardware cues unless explicitly in dry-run/mock mode.
- Do not call external providers unless feature flags and API keys are present.
- Do not hard-code any secrets.
- Do not expose public URLs or signed URLs.
- Do not bypass root-admin approval.
- Preserve all current tests.

Required environment variables:

ELEVENLABS_API_KEY
