Goal:
Create a root-admin approval board that allows productions to move through controlled internal approval stages based on readiness reports, asset completeness, sandbox validation, and audit history.

Important:
Do not enable real Unreal sends.
Do not enable real 4D hardware sends.
Do not enable publishing.
Do not expose secrets.
Do not create public URLs.
Do not auto-approve anything.
This is workflow, validation, UI, and audit only.

Scope:

1. Add new section in /admin/production-house:
Approval Board

Features:
- list productions by approval state
- show readiness score
- show blockers/warnings
- show asset completeness
- show latest sandbox statuses
- approve for internal review
- approve for Unreal sandbox
- approve for 4D sandbox
- mark as blocked
- request revision
- view approval history

2. Add approval stages:
- draft
- needs_review
- internal_review_approved
- unreal_sandbox_approved
- four_d_sandbox_approved
- blocked
- revision_requested

Do not add real_unreal_approved or real_4d_approved yet.

3. Add backend endpoints:
GET /api/admin/production-house/approval-board

POST /api/admin/production-house/approval-board/:productionId/transition
GET /api/admin/production-house/approval-board/:productionId/history

All routes must be root-admin gated.

4. Transition rules:
- draft → needs_review allowed

- any state → revision_requested allowed with required reason
- blocked → needs_review allowed with required reason
- revision_requested → needs_review allowed with required reason
