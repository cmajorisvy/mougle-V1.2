Next task: Add 4D Hardware Sandbox Contract and Local 4D Stub to Mougle AI Production House.

Goal:
Create a sandbox-only 4D hardware contract and local 4D stub that validates 4D cue payloads, simulates hardware responses, records dry-run history, and prepares the system for future physical 4D cinema integrations.

Important:
Do not connect to real 4D hardware.
Do not send DMX, OSC, UDP, MIDI, serial, relay, fog, wind, scent, vibration, motion-seat, or lighting commands.
Do not connect to physical devices.
Do not enable realSendAllowed.
Do not publish anything.
This is sandbox/dry-run only.

Scope:

1. Add new section in /admin/production-house:
4D Hardware Sandbox

Features:
- 4D sandbox health check
- supported effect types
- cue payload validator
- dry-run cue sender
- response viewer
- dry-run cue history
- safety badges:
  4D Sandbox
  Dry Run Only
  No Real Hardware
  No DMX Send
  No OSC/UDP Send
  No Motion Seat
  No Fog/Wind/Scent
  No Public Output

2. Add backend endpoints:
GET /api/admin/production-house/4d/sandbox/health
GET /api/admin/production-house/4d/sandbox/supported-effects
POST /api/admin/production-house/4d/sandbox/validate-cue
POST /api/admin/production-house/4d/sandbox/send
GET /api/admin/production-house/4d/sandbox/history

All routes must be root-admin gated.

- target

- dryRun is not true
- realSendAllowed is true

- effectType is unsupported

- durationMs is negative or too high

- visibility is not admin_only_internal
