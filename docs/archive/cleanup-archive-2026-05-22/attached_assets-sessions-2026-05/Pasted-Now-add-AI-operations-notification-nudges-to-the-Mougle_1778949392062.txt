Part A — Backend nudge endpoint

Add an admin-only endpoint:

GET /api/admin/ai-ops/notifications

Return a list of current AI ops nudges derived from existing data.

Suggested response shape:

{
  "notifications": [
    {
      "id": "stale-running-jobs",
      "severity": "info|warning|critical",
      "title": "Stale running jobs detected",
      "message": "3 AI jobs have been running longer than 15 minutes.",
      "count": 3,
      "actionLabel": "View jobs",
      "actionUrl": "/admin/ai-jobs?status=running",
      "createdAt": "...",
      "metadata": {}
    }
  ]
}

Suggested nudges:

1. Stale running jobs
   - warning if staleRunningJobs > 0
   - action: /admin/ai-jobs?status=running

2. Pending jobs but no online workers
   - critical if pendingJobs > 0 and onlineWorkers === 0
   - action: /admin/ai-workers

3. Failure spike
   - warning if failedJobsLast24h >= 10
   - critical if failedJobsLast24h >= 25
   - action: /admin/ai-jobs?status=failed

4. Cleanup overdue / large eligible cleanup
   - warning if cleanupEligibleCounts total > 1000
   - action: /admin/ai-retention

5. Workers unhealthy
   - warning if unhealthyWorkers > 0
   - action: /admin/ai-workers

6. No cleanup history
   - info if no cleanup has ever run
   - action: /admin/ai-retention

7. Last cleanup failed
   - warning or critical if lastCleanupStatus === "failed"
   - action: /admin/ai-retention

Implementation notes:
