Autopilot modes:
Create these modes:

1. manual
Human approval required for all render/playout actions.

2. autopilot_preview
System continuously generates packages/manifests but does not render or play.

3. autopilot_internal_playout
System can run 24/7 internally:
- generate newsroom script
- generate voice if provider enabled
- generate render manifest
- generate playout item
- play internally/admin-only
- no public publishing

4. autopilot_public_publish
Create schema/type placeholder only.
Must be disabled by default.
Must require future migration + legal/editorial approval.
Do not implement public publishing in this task.

Autopilot safety thresholds:
Autopilot may process only items that satisfy:
- status is verified or approved internal equivalent
- not rejected
- not disputed
- not correction unless marked correction-safe
- not developing unless setting allowDevelopingInternalOnly is true
- confidence score above configured threshold
- at least configured minimum source count
- no blocked media rights
- no missing required headline/script/source fields
- no safety flag violation

Blocked categories:
Autopilot must block or require manual review for:
- elections
- war/conflict escalation
- health/medical advice
- financial recommendations
- legal accusations
- death reports
- criminal allegations
- minors
- graphic violence
- disputed stories
- low-confidence stories
- insufficient sources
- rights-blocked media
