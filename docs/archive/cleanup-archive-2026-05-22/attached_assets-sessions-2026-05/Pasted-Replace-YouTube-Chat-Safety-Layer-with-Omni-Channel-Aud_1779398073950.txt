Replace “YouTube Chat Safety Layer” with “Omni-Channel Audience Safety Layer”.

Create:
server/services/omni-channel-audience-safety-service.ts

Goal:
Monitor, filter, classify, moderate, and safely route audience comments, live chats, gifts, superchats, tips, replies, mentions, and community messages across all legally connected platforms.

Supported channels:
- YouTube Live Chat / comments
- Facebook Live comments / page comments
- X / Twitter replies, mentions, spaces comments if API allows
- Telegram channel/group messages
- Instagram comments if API allows
- TikTok Live/comments if API allows
- LinkedIn comments if API allows
- Reddit comments if API allows
- future platform adapters

Important:
Only use official APIs, approved webhooks, or legally permitted integrations.
Do not scrape platforms where prohibited.
Do not bypass rate limits.
Do not collect private messages unless the user/account explicitly authorized it.
Do not expose raw user PII on newsroom screens.
Do not let the robot/anchor read abusive, spam, private, or misinformation content.

5. Omni-channel rules:

Block from anchor/robot speech and screen display:
- spam
- abuse
- hate
- harassment
- threats
- PII
- doxxing
- misinformation
- unsupported breaking claims
- copyrighted text dumps
- malicious links
- impersonation
- scams
- platform policy violations

Gift/superchat/tip handling:
- acknowledge only if message is safe
- never prioritize unsafe paid messages
- no playful gift reactions during sensitive stories
- no gift celebration overlays during disaster/war/death/crime/medical/children/active crisis stories

Sensitive-story override:
For sensitive stories, allow only:
- neutral acknowledgments
- verified questions
- moderator-reviewed comments
- no jokes
- no shapeshift mode
- no aggressive robot gestures
- no celebratory animations

6. Platform action rules:

The system may only edit/delete/hide/reply if:
- the platform API officially allows it
- the connector permission is present
- the message belongs to a connected account/page/channel where Mougle has rights
- the action passed safety validation
- approvalStatus is approved where required
- rate limits allow it

If permission is missing:
- create AudienceModerationCommand with commandAllowed:false
- set blockerReason
- do not call platform API

In this phase:
- platformSendAllowed:false by default
- all moderation commands are simulation_only unless explicitly approved later
