Build Mougle Neural Newsroom Automation + Broadcast-Grade Virtual Screen Director.

Goal:
Create a fully automated but safety-constrained orchestration layer for Mougle’s 24/7 AI newsroom.

The system must connect:
- raw news feeds
- verified newsroom data
- BroadcastBriefs
- LegalEventVisualPlans
- NewsroomScreenData
- AI anchor
- robot explainer / robot screen director
- all newsroom displays
- ticker
- lower-third
- source/confidence panels
- claims/timeline panels
- event display wall
- YouTube chat/gifts/spam safety
- 4D cue simulation
- Preview Studio

The robot anchor should behave like a precise automatic broadcast screen director while another anchor reads the news. It can select verified sources, route visuals, zoom/crop, trigger fullscreen displays, move anchor to corner mode, restore the world map, and point to claims/sources/timeline — but only through validated simulation records.

Hard rules:
- no copyrighted footage reuse
- no logo/watermark removal
- no unauthorized rebroadcast
- no private admin data on screens
- no YouTube/social posting without approval
- no direct Spyder/Barco/Novastar command in this phase
- no real LED processor command
- no live Unreal execution
- no Movie Render Queue
- no Sequencer
- no real 4D hardware
- no publishing
- all outputs draft/internal-only unless separately approved

Architecture:
Raw News Feeds
→ Source Trust / Verification
→ BroadcastBrief
→ LegalEventVisualPlan
→ NewsroomScreenData
→ ApexLoad Optimizer
→ PreCognition Planner
→ FlowState Conductor
→ Virtual Screen Director
→ Broadcast-Grade Screen Safety
→ Preview Studio / Simulation
→ Approval / Future Gateway

The final authority must be deterministic safety validation, not free AI generation.

------------------------------------------------------------
1. ApexLoad Optimizer
------------------------------------------------------------

Create:
server/services/apexload-newsroom-optimizer.ts

Purpose:
Score every incoming story and decide how much production effort it deserves.

Inputs:
- article
- sourceReliability
- verificationConfidence
- impactScore
- freshness
- region
- publicInterest
- visualPotential
- rightsReadiness

Output:
ApexLoadDecision {
  decisionId,
  storyId,
  apexScore,
  productionTier:
    - text_only
    - voice_summary
    - newsroom_read
    - full_visual_package
    - cinematic_4d_treatment
  reasonCodes[],
  costEstimate,
  approvalStatus:"draft",
  visibility:"admin_only_internal",
  publicUrl:null,
  signedUrl:null,
  realSendAllowed:false,
  executionEnabled:false,
  safetyEnvelope
}

Formula:
apexScore =
0.25*impactScore +
0.20*sourceReliability +
0.15*verificationConfidence +
0.10*freshnessScore +
0.10*regionalImportance +
0.10*publicInterest +
0.05*visualPotential +
0.05*rightsReadiness

Tier routing:
- < 40: text_only
- 40–60: voice_summary
- 60–75: newsroom_read
- 75–90: full_visual_package
- 90+: cinematic_4d_treatment

6. Confidence Vector
------------------------------------------------------------

For every automatic action calculate:

C_total = min(
  C_source,
  C_verification,
  C_license,
  C_screen_match,
  C_sensitivity,
  C_chat_safety,
  C_fallback
)

Rules:
- C_total >= 0.90: automatic simulation allowed
- 0.75 <= C_total < 0.90: assisted preview only
- 0.50 <= C_total < 0.75: admin review required
- C_total < 0.50: reject and fallback

Important:
Use minimum confidence, not average. A story can be verified while the media source is still unsafe.

12. Virtual Screen Routing Behavior
------------------------------------------------------------

For disaster/hurricane story:
- MGL_BACK_DISPLAY_Main shows satellite/map fly-in
- MGL_EVENT_DISPLAY_Fullscreen takes full wall only if source is legal and approved
- anchor moves to MGL_ANCHOR_CORNER_Left or MGL_ANCHOR_CORNER_Right
- robot points at storm path and impact metrics
- source panel shows NOAA/NASA/public-domain/licensed sources
- claims panel shows verified claims
- timeline panel shows event timeline
- ticker shows safety updates
- 4D cue timeline suggests alert pulse/light flash only as draft

For AI/technology story:
- big wall shows AI data visualization
- robot points at chip/brain/network diagram
- anchor remains desk or side presenter
- side panels show source verification and model performance

For debate/podcast:
- background switches to discussion wall
- source/confidence panels stay visible
- lower-third and ticker remain active
