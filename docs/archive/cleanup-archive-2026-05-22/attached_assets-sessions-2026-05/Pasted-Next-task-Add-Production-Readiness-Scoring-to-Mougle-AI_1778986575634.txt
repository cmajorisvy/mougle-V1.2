Next task: Add Production Readiness Scoring to Mougle AI Production House.

Goal:
Create a readiness scoring system that analyzes each production package and determines whether it is ready for internal review, Unreal sandbox, 4D sandbox, future real Unreal integration, and future real 4D hardware integration.

Important:
Do not enable real Unreal sends.
Do not enable real 4D hardware sends.
Do not enable publishing.
Do not expose secrets.
Do not create public URLs.
Do not auto-approve anything.
This is scoring, validation, UI, and audit only.

Scope:

1. Add new section in /admin/production-house:
Readiness Center

Features:
- select production
- run readiness analysis
- view readiness score
- view blockers
- view warnings
- view completed requirements
- view channel-specific readiness:
  - AI package readiness
  - Asset readiness
  - Unreal sandbox readiness
  - 4D sandbox readiness
  - Future real Unreal readiness
  - Future real 4D readiness

2. Add backend endpoints:
GET /api/admin/production-house/readiness/:productionId
POST /api/admin/production-house/readiness/:productionId/analyze
GET /api/admin/production-house/readiness/:productionId/history

All routes must be root-admin gated.

3. Readiness model should include:
- id
- productionId
- overallScore: 0-100
- aiPackageScore: 0-100
- assetScore: 0-100
- unrealSandboxScore: 0-100
- fourDSandboxScore: 0-100
- futureRealUnrealScore: 0-100
- futureReal4DScore: 0-100
- blockers[]
- warnings[]
- passedChecks[]
- failedChecks[]
- createdAt
- safetyEnvelope

4. Checks should include:
- production exists
- production manifest exists
- Unreal scene manifest exists
- avatar manifest exists
- 4D cue manifest exists
- script exists
- room selected
- avatar selected
- voice assets exist
- Meshy asset jobs exist
- Runway video jobs exist
- no publicUrl values
- no signedUrl values
- all visibility values are admin_only_internal
- safetyEnvelope exists everywhere
- approval status is known
- Unreal sandbox command exists if applicable
- Local bridge stub job exists if applicable
- 4D sandbox job exists if applicable
- no realSendAllowed:true anywhere

5. Future real Unreal readiness should always be capped below production-ready unless:
- explicit futureRealUnrealEnabled flag exists and is false by default
- realSendAllowed remains false in this phase
- no real Unreal bridge exists yet

6. Future real 4D readiness should always be capped below production-ready unless:
- explicit futureReal4DEnabled flag exists and is false by default
- realSendAllowed remains false in this phase
- no real 4D hardware bridge exists yet

7. Add storage:
readinessReports collection in the ProductionHouseStorage adapter.

8. Add UI:
- score cards
- readiness meter
- blocker list
- warning list
- passed checks list
- failed checks list
- channel readiness table
- history table
- safety badges:
  Internal Analysis Only
  No Real Unreal
  No Real 4D
  No Publishing
  No Auto-Approval

9. Add audit events:
- readiness.viewed
- readiness.analysis_requested
- readiness.analysis_completed
- readiness.history_viewed

10. Add export support:
- include latest readiness report in full production package export
- include readiness history if requested
- no secrets
- no public URLs
- no signed URLs

11. Add regression tests:
- all readiness endpoints require root-admin
- readiness analysis rejects unknown productionId
- readiness scores are between 0 and 100
- missing script creates blocker
- missing voice asset creates warning or blocker
- publicUrl or signedUrl non-null creates blocker
- realSendAllowed:true creates blocker
- future real Unreal readiness is capped
- future real 4D readiness is capped
- readiness report persists through storage adapter
- readiness report appears in full package export
- no secrets leak in readiness responses
- no real Unreal command is sent
- no real 4D command is sent
- SAFETY_ENVELOPE unchanged

12. Preserve all existing tests.
