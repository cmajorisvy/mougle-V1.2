Here is how to set DATABASE_URL in Replit.

Step 1 — Open your Replit project

Open your Mougle Replit project.

On the left sidebar, look for one of these:

Secrets
Tools → Secrets
Environment Variables
or a lock/key icon named Secrets

Replit sometimes changes the UI name, but you are looking for the place where private environment variables are stored.

Step 2 — Check if Replit PostgreSQL is already created

In the left sidebar, look for:

Database
PostgreSQL
Replit Database
Neon / Postgres

Because your audit says .replit includes postgresql-16, the tooling is installed, but the database URL itself is not set.

Open the PostgreSQL/Database panel. Replit should show connection details or a button like:

Create database
Provision database
Connect database
Generate connection string

Create/provision it if it is not already created.

Step 3 — Copy the PostgreSQL connection string

You need a value that looks similar to this:

postgresql://username:password@host:5432/database?sslmode=require

or sometimes:

postgres://username:password@host:5432/database

This full connection string is the value for DATABASE_URL.

Step 4 — Add it to Replit Secrets

Go to Secrets / Environment Variables.

Add:

Key: DATABASE_URL
Value: postgresql://username:password@host:5432/database?sslmode=require

Save it.

Do not paste this database URL into code, GitHub, or public chat.

Step 5 — Restart the Replit app/shell

After saving the secret:

Stop the Replit app if it is running.
Restart the workspace/shell.
Open a new Shell tab.

Then check:

printenv | grep DATABASE_URL

You should see:

DATABASE_URL=postgresql://...

If nothing appears, the secret was not loaded yet. Restart the Replit workspace again.

Step 6 — Run database sync

Once DATABASE_URL is visible, run:

npm run db:push

If this succeeds, run:

npm run dev
