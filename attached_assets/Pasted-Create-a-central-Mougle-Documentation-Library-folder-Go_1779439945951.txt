Create a central Mougle Documentation Library folder.

Goal:
All project documents must be organized under one clear docs/library/ structure so PDFs, Markdown files, text/notepad files, Word-style docs, prompts, reports, designs, runbooks, archives, audits, and future task documents are easy to find and reuse.

This is documentation organization only.

Do not change source code.
Do not change routes.
Do not change services.
Do not change schemas.
Do not run migrations.
Do not change dashboard behavior.
Do not delete documents.
Do not permanently remove anything.

docs/library/

Rules:
- Do not move active docs that are referenced by replit.md unless you update the link safely.
- Do not break existing links.
- Do not move archive files without preserving archive paths.
- Do not move backup manifests unless copied, because backup paths are important.
- Do not move reports used by T1–T5 unless creating aliases/pointers.
- Prefer index-first, move-later.
